# AI 量化交易系统技术方案

> 版本：v2.0
> 日期：2026-07-11
> 定位：个人使用、**加密货币现货**、AI 辅助、人可控、新手友好
> 一句话目标：**先用单个简单策略跑通"生成→回测→dry-run→小资金实盘"闭环；验证有效后，再把多流派共识作为进阶选项叠加。全程只做现货（不碰合约/杠杆/做空），守住"研究层与执行层"的边界。**
>
> v2.0 重大调整（基于反过拟合审查）：
> 1. **锁定现货**：删除做空、资金费率过滤、套利流派（这些是合约概念，现货用不上）。
> 2. **共识降级为进阶可选**：先跑通单策略验证有效，再考虑叠加共识——避免一上来就建复杂系统、引入大量待调参数造成设计层过拟合。
> 3. **新增可观测性 + LLM 日报**（4.5）：解决"长期不交易怎么看效果"。
> 4. 修正：仓位模型、hyperopt 过拟合警告、路线图与架构对齐。

## 0. 阅读前必读：三条底层原则

这份方案不是"装个软件就赚钱"的教程。它建立在前期调研得出的三条硬结论上：

1. **工具不赚钱，纪律和验证才决定生死。**（强判断，有 AI-Trader 官方论文佐证：GPT 交易 3378 次亏 63%，Claude 交易 21 次基本不亏，差别在纪律不在模型）
2. **AI 负责"想和判断"，成熟执行器负责"守纪律下单"。研究代码永远不和实盘资金共用一条路径。**（借鉴 QuantDinger 架构的核心设计原则）
3. **挑策略要挑"最稳健的"，不是"回测最赚的"。回测最赚的往往是最会骗你的（过拟合）。**
4. **从最简单开始，验证有效再加复杂度。**（反过拟合审查得出）每多一个参数/流派/旋钮，就多一个过拟合表面。一个 10 参数的共识系统比一个 2 参数的均线策略**更容易**过拟合。所以：先跑通单策略，简单的不够用了，再叠加共识。

> ⚠️ 风险声明：加密货币量化交易属于高风险行为。本方案所有"预期收益"均为**待验证**，历史回测不代表未来。方案的价值是"帮你少犯错、少爆仓、看清楚"，不是"保证盈利"。

## 1. 你的原始设想 vs 修正后的设想

### 1.1 你的原始设想

```
聊天想几个策略 → 自动 dry-run → 挑最好的 → 实盘
```

### 1.2 修正后的设想（现货 + 分阶段，简单先行）

**主线（先做，必须跑通）：单策略闭环**
```
聊天生成 1 个简单现货策略（如趋势跟踪）
        ↓
回测（多币种+多时段）→ 看5个体检指标
        ↓
dry-run 模拟盘长跑
        ↓
按稳健度判断是否可用（不看收益看稳健）
        ↓
小资金现货实盘（$100-500）
        ↓
Freqtrade 纪律执行 + Telegram 通知 + LLM 日报观测
```

**进阶（可选，主线验证有效后才考虑）：多流派共识**
```
若单策略不够用 → 叠加 3 个正交流派（趋势/均值回归/量价）
        ↓
共识投票 + 死区门槛（"不交易"是常态）
        ↓
共识 → 现货买入/持有/卖出（无做空）
```

**核心变化**：
1. **现货 only**：不做空、不加杠杆、不碰合约。最坏结果是亏本金，不会爆仓。
2. **简单先行**：单策略是主线，共识是进阶可选，避免一上来就过拟合。
3. **"不交易"是合法且最常见的结果**（见 4.3）。

> ⚠️ 现货约束带来的删减：**做空、资金费率过滤、套利流派**都是合约概念，现货用不上，本版已全部移除。这让方案更简单，正符合原则4。

## 2. 系统总体架构

采用**三层分离**架构（研究层 / 执行层 / 数据层），三层不共用代码路径，研究永远碰不到真钱，除非你显式"晋级"一个策略到实盘。

```
┌──────────────────────────────────────────────────────────┐
│  第一层：研究层（AI 大脑，可以模糊、可以多视角）                │
│  ─────────────────────────────────────────────────────    │
│  • 聊天生成多流派策略（LLM + 策略模板）                        │
│  • 多策略 dry-run 对比（稳健度排序）                          │
│  • 共识/分歧计算 → 输出置信度                                 │
│  产物：策略配置 + 信号 + 置信度（绝不直接下单）                 │
└───────────────────────────┬──────────────────────────────┘
                            ↓ 只传"策略配置/信号"，显式晋级才进执行层
┌──────────────────────────────────────────────────────────┐
│  第二层：执行层（Freqtrade，必须确定、必须有纪律）              │
│  ─────────────────────────────────────────────────────    │
│  • dry-run 模拟盘（默认，长期跑）                             │
│  • 实盘执行（双开关解锁，默认关闭）                            │
│  • 硬性风控：止损 / 熔断 / 冷却 / 仓位上限                     │
│  • Telegram bot 指挥 + 通知（原生）                          │
└───────────────────────────┬──────────────────────────────┘
                            ↓ 辅助过滤（缓存，非实时）
┌──────────────────────────────────────────────────────────┐
│  第三层：数据层（便宜的实时拉，贵的缓存）                       │
│  ─────────────────────────────────────────────────────    │
│  • 行情：交易所现货 API（免费）                               │
│  • 情绪：恐惧贪婪指数（免费，每天缓存1次，可选）               │
│  • LLM 判读/日报：批量预生成，缓存复用（省钱，学 stockgod）    │
└──────────────────────────────────────────────────────────┘
```

> **关键**：你的 WSL snap docker 在拉 GHCR 大镜像时会崩溃（已实测验证）。**Freqtrade 用 pip 直接装，完全绕开这个坑。** 这是选它的现实理由，不只是它更好。

### 3.3 为什么不用 Vibe-Trading（决策说明）

前期对话重点讨论过 Vibe-Trading，但 v2.0 决定**不采用**，理由：

| Vibe-Trading 原职责 | 现在谁接手 | 说明 |
|---------------------|-----------|------|
| 聊天生成策略 | LLM 直连 | 生成策略本质也是调 LLM，直接调等于跳过中间商，且能让 LLM 直接产出 Freqtrade 格式，少一次转换 |
| 回测 | Freqtrade 内置 | 避免"两个回测引擎结果不一致"制造的回测骗局风险 |

**核心理由（呼应原则4）**：每多一个组件 = 多一层依赖、多一个坑、多一层调试难度。Vibe-Trading 在这里是可以打薄的中间层，去掉它链路更短更可控。

**代价（诚实说）**：丢了 Vibe-Trading 的 452 因子库、可视化界面、一站式体验。但这些对**新手单策略主线**非必需（基础指标 LLM 都会写，Freqtrade 也有 Web UI）。等进阶到需要大量因子库时可再评估引入。

## 4. 分层详细设计

### 4.1 执行层（Freqtrade）— 纪律怎么强制

纪律不靠自觉，靠代码强制。三层硬约束：

```python
# 第1层：策略级止损（每笔交易，无条件执行）
stoploss = -0.05                    # 亏5%自动砍，不给"再等等"的机会

# 第2层：Protections 账户级熔断（默认要手动开启！）
protections = [
    {"method": "StoplossGuard",      # 短期内多次止损 → 停止开新仓
     "lookback_period_candles": 24,
     "trade_limit": 3,
     "stop_duration_candles": 12},
    {"method": "MaxDrawdown",        # 回撤超阈值 → 全面停手
     "max_allowed_drawdown": 0.10},
    {"method": "CooldownPeriod",     # 刚亏完 → 强制冷却
     "stop_duration_candles": 6},
]

# 第3层：仓位约束
max_open_trades = 3                 # 最多同时3个仓，防梭哈
stake_amount = 100                  # 每笔固定金额(USDT)，防越亏越加码
# 说明：新手先用"固定金额/笔"，简单可控。别用共识分动态调仓位——
# 那是进阶功能，会引入更多待调参数（原则4）。
```

**仓位模型（现货、新手版）**：
- 每笔固定金额（如 100 USDT），最多 3 个并发仓位
- 单笔风险 = 止损幅度 × 单笔金额（如 100 × 5% = 最多亏 5 USDT/笔）
- **先不做"按共识分动态调仓位"**——那是进阶功能，等主线跑通再说

| 人性弱点 | Freqtrade 如何强制治理 |
|----------|----------------------|
| 亏了死扛 | 到止损线自动砍 |
| 连亏后想回本、加倍下注 | StoplossGuard 停手 |
| 大跌恐慌乱操作 | MaxDrawdown 熔断 |
| 手痒频繁交易 | CooldownPeriod 冷却 |

### 4.2 研究层 — 你不懂策略，怎么生成 + 怎么判断

#### （A）生成：大白话 → 多流派策略

关键：不生成同质的策略变体（那样"分歧"是假的），要生成**维度真正正交（互不相关）的流派**。

**⚠️ 正交性检查（v1.1 修正的重点）**：流派之间必须回答"不同的问题"，否则投票失衡。反例——"趋势"和"突破"本质都是"顺势/动量"，放一起会让投票变成"顺势票远多于逆势票"，共识失去制衡意义。

推荐的 **3 个正交流派**（奇数，天然避免平票）：

| 流派 | 回答的问题 | 方向性 | 赚钱行情 |
|------|-----------|--------|---------|
| 趋势（均线/MACD） | 价格在朝哪个方向走？ | 顺势 | 单边行情 |
| 均值回归（RSI/布林带） | 价格是不是偏离过头了？ | 逆势 | 震荡行情 |
| 量价（OBV/量能） | 这个价格动作背后有没有真金白银在推？ | 正交（与方向无关） | 确认真假突破 |

> 说明：**突破合并进趋势**（两者高度相关）；**套利（资金费率）不投票**，降级为"情绪过滤器"（见 4.4）。3 个正交流派 = 顺势 + 逆势 + 独立确认，才是真正的多视角制衡。

对 LLM 的标准提示词模板：

```
我不懂量化，帮我生成一个可以放进 Freqtrade 的 [流派] 策略：
- 品种：BTC/USDT
- 入场：[大白话描述，如"跌多了想抄底"→ 自动翻译成 RSI<30]
- 出场：[如"涨10%走 或 跌破某线"]
- 止损：亏5%必须走
- 时间周期：以4小时为锚定决策周期（可引用日线/1小时数据辅助）
- 输出：完整 IStrategy 类代码 + 该策略的适用行情说明
```

#### （B）判断：不懂策略也能看的 5 个体检指标 + 3 个防坑问题

**5 个体检指标（红绿灯）：**

| 指标 | 问 AI 要 | 🟢 好 | 🔴 危险 |
|------|---------|-------|---------|
| 最大回撤 | 最多亏过多少 | <20% | >40% |
| 胜率 | 100次赢几次 | 40-60% | >70%（可疑！过拟合信号） |
| 夏普比率 | 夏普多少 | >1 | <0.5 |
| 交易次数 | 回测期交易几次 | 适中 | 几百上千（手续费吃光） |
| 盈亏比 | 赢时赚多少vs输时亏多少 | 赢>输 | 赢<输 |

> ⚠️ 反直觉但救命：**胜率 >70% 要警惕，不是高兴。** 真实市场没那么好赚，超高胜率通常=过拟合，或"赢很多次小的、输一次大的"（迟早爆仓）。

**3 个防坑问题（筛掉 90% 垃圾策略）：**

1. **"换 ETH/SOL 还赚吗？"** — 只在 BTC 好 = 过拟合，扔掉
2. **"2022熊市和2023牛市分开测各是什么表现？"** — 只在某段好 = 靠运气
3. **"手续费0.1%+滑点0.05%扣完还赚吗？"** — 扣完就亏 = 假象

### 4.3 研究层 — 多流派共识（🔶 进阶可选，主线验证后再上）

> **重要**：本节是**进阶可选**功能。**不要一开始就做。** 先按主线跑通单策略（4.2），确认"生成→回测→dry-run→实盘"闭环能走通、单策略确实有效之后，若发现单策略不够稳健，再考虑叠加共识。一上来就建共识系统会引入大量待调参数，是设计层过拟合的主要来源（原则4）。

#### （A）时机同步：单策略 + 锚定决策周期（关键实现）

多流派最容易踩的坑是**时机错位**：不同流派天然属于不同周期，若用多个独立实例再外部聚合，会出现"有的信号新、有的已过期"，且无法整体回测。

**✅ 正确实现**：用**一个** Freqtrade 策略，设定**一个锚定决策周期**（如 4h）。每根锚定 K 线收盘时，**一次性**算出所有流派的子信号，再投票：

```
单个 Freqtrade 策略，锚定决策周期 = 4h：
  每根 4h K线收盘时，一次性算出：
    ├ 趋势子信号   （可引用 4h + 日线数据）
    ├ 均值回归子信号（可引用 1h + 4h 数据）
    └ 量价子信号   （成交量确认）
  → 加权投票 → 共识 → 决策
```

**收益**：
- 所有子信号在同一时刻算完，不存在"等最慢流派" → **延迟锁定 = 锚定周期（4h），不会被放大**
- 整体可回测、可复现（仍是同一引擎）
- 各流派仍能看多周期数据（Freqtrade 的 informative pairs 机制），但**决策统一在锚定周期发生**

> ❌ 反模式：多个策略跑多个不同周期实例再外部聚合 → 信号 schema 不同步、无法整体回测、需自写同步层，易错。

#### （B）"不交易"是合法且最常见的决策

**核心原则：不是每个周期都要交易。"不动"往往是最优决策。**

一个健康策略，绝大多数周期应该是"不动"的。现货场景下，每个决策周期共识投票有**四种合法结果**：

| 结果 | 触发条件 | 动作 |
|------|---------|------|
| 买入/加仓 | 强共识看多（超过门槛） | 买入 |
| 卖出/减仓 | 强共识看空（超过门槛） | 卖出持有的现货 |
| **不交易（分歧）** | 流派分歧大、共识落在"死区" | **什么都不做** |
| **不交易（无信号）** | 没有流派给出明确信号 | **什么都不做** |

> 注意：现货**只有"买入/持有/卖出"三态，没有做空**。共识看空 = 卖出手里的币或不买，不是开空单。

后两者（不交易）是**常态**。（AI-Trader 论文佐证：GPT 交易 3378 次亏 63%＝"每次都想动"；Claude 交易 21 次基本不亏＝"大多数时候不动"。少动是特征，不是 bug。）

#### （C）共识决策表（现货版，含"死区"门槛）

不能只看"多数票"，必须设**死区**，否则会在临界点频繁抖动、反复交易。以 3 个正交流派、每个投 +1（看多）/ 0（中立）/ -1（看空）为例，加权得共识分（范围 -1 ~ +1）：

| 共识分区间 | 含义 | 现货决策 |
|-----------|------|---------|
| ≥ +0.6 | 强共识看多 | 买入（若空仓）/ 持有（若已持仓） |
| -0.6 ~ +0.6 | 分歧/弱信号 | **不交易**（死区，维持现状） |
| ≤ -0.6 | 强共识看空 | 卖出持有的币（若持仓）/ 继续空仓 |

> **死区（-0.6 ~ +0.6）就是"不交易区"**，把"不是每次都交易"变成可执行规则。阈值（0.6）是**待验证**参数，需 dry-run 调优。**警告：不要用 hyperopt 在历史数据上暴力优化这个阈值——那是过拟合温床（见第7节）。**

#### （D）两个连带简化

1. **平票不再是问题**：落在死区 → 结果就是"不交易"，无需强行设"决胜票"。
2. **持仓不变 = 不交易**：已持仓且共识仍看多 → 继续拿着，**不重复下单、不卖了重买**（避免白磨手续费）。

### 4.4 数据层 — 情绪监控（现货版，务实，别过度）

现货场景删掉了资金费率（那是永续合约概念）。只保留一个客观、免费、够用的情绪指标：

| 层次 | 数据源 | 角色 | 用不用 |
|------|--------|------|--------|
| 市场情绪指数 | alternative.me 恐惧贪婪指数（免费） | 仓位微调（可选） | ✅ 用 |
| 链上数据 | 交易所净流入/巨鲸 | 进阶信号 | ⚠️ 别急，进阶再说 |
| 资金费率 | — | 合约概念，现货无 | ❌ 不用 |
| 新闻/社交情感 | 推特 NLP | — | ❌ 别碰，噪音大，散户做不出稳定alpha |

用法（简单规则，且是**可选**的，主线阶段可以先不加）：
```
恐惧贪婪指数 < 20（极度恐惧） → 别人恐惧我贪婪，可略提高置信/仓位
恐惧贪婪指数 > 80（极度贪婪） → 别人贪婪我谨慎，可略降仓位
```

> **成本铁律（学 stockgod）**：恐惧贪婪指数每天才更新1次（每天缓存1次即可）；只有行情 OHLCV 才需要每周期实时拉；LLM 判读/日报是每天1次的批量任务。别用 LLM 实时解读每根 K 线——烧钱且没必要。

### 4.5 可观测性 — 长期不交易时如何评估（关键补充）

因为"不交易"是常态（4.3），你会遇到"长期没交易，看不出效果"的困惑。解法是**监控"决策过程"，而不是等"交易结果"**。

#### （A）先分清两种"不交易"

| 类型 | 含义 | 好坏 |
|------|------|------|
| 健康的等待 | 市场确实没机会，策略明智空仓 | ✅ 好 |
| 病态的沉默 | 阈值设太严 / 流派逻辑错 / 数据没进来 | ❌ 坏（伪装成纪律） |

**看不到过程就无法区分这两者**，所以必须记录决策过程。

#### （B）决策过程日志（每周期都记，不管是否交易）

```
每根决策 K线收盘记一行：
时间 | 各流派票 | 共识分 | 落在哪个区 | 决策 | 距触发差多少

示例：
07-10 12:00 | 趋势+1 回归0 量价+1 | +0.53 | 死区 | 不交易 | 差0.07触发买入
07-10 16:00 | 趋势+1 回归+1 量价+1 | +0.87 | 强多 | 买入   | ✓触发
```
（注：单策略主线阶段，记录该策略自身的信号值和触发距离即可，不需要共识分。）

#### （C）五个"非盈亏"健康指标

| 指标 | 健康信号 |
|------|----------|
| 信号/共识分时间序列 | 有正常波动，不是一直贴 0 |
| 触发接近度 | 偶尔接近阈值（说明系统活着、非死板） |
| 各流派投票分布 | 每个流派都偶尔发声，不永远沉默 |
| 数据新鲜度 | K线时间戳在正常更新（排除卡死） |
| 回测频率基准 | 当前交易频率 ≈ 该策略历史回测频率 |

#### （D）长期不交易的三种诊断

| 现象 | 诊断 | 处理 |
|------|------|------|
| 共识分老差一点触发 | 阈值太严 | 正常调参（降死区），重新 dry-run |
| 某流派从不发声 | 逻辑/数据 bug | 去查那个流派（dry-run 的价值） |
| 共识有波动但没到极端 | 行情真没机会 | 用回测验证：历史有机会时它会不会交易 |

#### （E）用回测放大样本（补 dry-run 样本太少）

低频 + 死区 + 共识 = 实盘交易很少 = 样本太少得不出结论。解法：**用回测把样本放大**。
```
dry-run（前进）：真实但慢、样本少
     + 
回测（回看2-3年）：快速得到几百个决策样本
     → 看历史交易频率/胜率/回撤
     → 判断"当前不交易"是常态还是异常
```
若回测显示"这策略过去2年也就交易20次"，那 dry-run 一个月没交易**完全正常**——它本就是低频策略。

### 4.6 LLM 日报 — 让 AI 帮你观测和汇报（安全用法）

这是 LLM 在本系统里**最合适、最安全的用法**：做"解读和汇报"，不碰"下单和改参数"。

#### （A）边界（铁律）

| LLM 该做 | LLM 不该做 |
|----------|-----------|
| ✅ 读决策日志，用人话解释"为什么这段没交易" | ❌ 实时决定买卖 |
| ✅ 判断系统"健康等待"还是"可能坏了" | ❌ 自动改参数/阈值 |
| ✅ 发现异常（某流派一直沉默） | ❌ 盘中干预 |
| ✅ 生成每日/每周汇报 + 建议 | ❌ 替你做决策 |

> ⚠️ LLM 日报里的建议（如"建议把阈值从0.6降到0.5"）是**给你看的参考，必须你确认后手动改**，绝不能让 LLM 自动改。否则破坏"研究层不碰执行"的边界。

#### （B）架构（每天1次，成本可忽略）

```
每天固定时间（收盘后）触发一次：
① 本地采集当天数据（便宜）：决策日志 / 持仓盈亏 / 恐惧贪婪指数
        ↓
② 喂给 LLM（贵，但每天只1次，用固定模板 prompt）
        ↓
③ LLM 输出结构化日报
        ↓
④ 推送到 Telegram（复用 Freqtrade 的 bot）
```
成本：每天 1 次调用，一个月约 30 次，几乎可忽略。

#### （C）日报模板

```
📊 量化系统日报 · 2026-07-10

【一句话结论】今日 6 个决策周期均未交易，系统健康，处于观望期。

【系统健康度】✅ 正常
- 数据更新：正常（K线时间戳最新）
- 信号活跃度：策略在正常计算，未达触发线
- 信号分范围：+0.2 ~ +0.53（有正常波动）

【为什么没交易】
今日信号最高 0.53，差 0.07 触发。属于"未形成足够强信号"的
健康等待，不是故障。

【值得注意】
- 16:00 一度到 0.53，是今日最接近触发的时刻
- 恐惧贪婪指数 22（恐惧区）

【建议（仅供参考，需你确认）】
无需操作。若连续 2 周信号都卡在 0.5 附近触发不了，可复盘
阈值是否偏严（但现在样本少，先观察）。

【持仓】无
【累计：本周交易 0 笔 | 本月 3 笔】
```

> 进阶：周报可做诊断——对比本周 vs 上周信号分布、分析已发生交易哪些像运气、读回测判断当前频率是否正常。**但同样只汇报不决策。**

## 5. 安全模型（实盘前必须落实的五道闸）

借鉴 QuantDinger 的安全设计，这是"AI/bot 参与交易"的安全底线：

1. **默认 paper-only** — dry-run 是默认状态，实盘要显式开
2. **实盘双开关** — 配置项 + 环境变量两处都打开才生效

## 6. 落地路线图（分阶段，每阶段有验证门槛）

> 原则：**主线（阶段0-4）先跑通单策略闭环。共识（阶段5）是进阶可选，主线验证有效才做。**

### 阶段 0：搭执行地基
```bash
# 用 pip 装 Freqtrade（避开 docker 坑），走你现有代理
export HTTPS_PROXY=http://127.0.0.1:7897
pip install freqtrade
freqtrade create-userdir --userdir user_data
freqtrade new-config --config config.json    # 现货、dry_run=true
```
**验证门槛**：`freqtrade trade --dry-run` 能跑起来，Telegram 能收到通知。

### 阶段 1：跑通第一个策略（体验"生成→判断"）
- 用 LLM 生成 **1 个**最简单的现货策略（如双均线趋势）
- 放进 Freqtrade 回测（多币种+多时段）
- 对照 5 个体检指标读结果

**验证门槛**：你能独立看懂"这个策略回撤多少、胜率多少、扣费后赚不赚"。

### 阶段 2：单策略稳健性验证
- 把这个策略在多币种（BTC/ETH）、多时段（牛/熊/震荡）回测
- 用 3 个防坑问题筛它
- **不急着加第二个策略**

**验证门槛**：这个单策略"换币种、换时段都不崩"（不要求都赚，要求不崩）。

### 阶段 3：dry-run 长跑 + 可观测性（关键，别跳过）
- 选中的单策略在模拟盘跑 **1-3 个月**
- 搭好决策日志 + LLM 日报（4.5/4.6），每天看健康度
- 记录：模拟盘表现是否接近回测（差距大 = 过拟合）

**验证门槛**：模拟盘实际表现 ≈ 回测表现，且你能接受它的回撤；日报能正常反映系统健康。

### 阶段 4：小资金现货实盘
- 双开关解锁实盘，币安现货 **$100-500**
- API Key 只开现货交易、禁提现、绑 IP
- 严格风控：每笔固定金额、日亏损上限
- 用回测样本 + 实盘共同判断（别只靠实盘 50 笔，样本不够）

**验证门槛**：实盘表现接近 dry-run，才考虑加码。

### 阶段 5（🔶 进阶可选）：叠加多流派共识
**前提：阶段1-4 已跑通，且单策略确实不够稳健时才做。**
- 在单个策略里加入 3 个正交流派（趋势/均值回归/量价）
- 实现共识投票 + 死区门槛（4.3）
- 重新走阶段 2-4 的验证流程

**验证门槛**：共识版在回测中稳健性**优于**单策略（否则回退——复杂没带来收益就是过拟合）。

## 7. 关键风险与应对

| 风险 | 应对 |
|------|------|
| 过拟合（最大陷阱） | 多币种+多时段验证、样本外测试、挑稳健不挑最赚、**先简单后复杂** |
| **hyperopt 暴力调参 = 过拟合温床** | ⚠️ 慎用 Freqtrade 的 hyperopt。在历史数据上优化出的"完美参数"实盘常失效。若用，必须留出**样本外时段**验证，且参数越少越好 |
| **设计层过拟合（功能堆积）** | 每加一个流派/参数/旋钮都要问"它验证过吗、比不加强吗"，不强就删 |
| 手续费/滑点吃光利润 | 回测按0.1%手续费+0.05%滑点算，控制交易频率 |
| 低频导致样本不足 | 用回测放大样本（4.5E），别只靠实盘等 50 笔 |
| dockerd 环境崩溃 | 全程用 pip 方案，不碰 GHCR 大镜像 |
| LLM API 成本失控 | 判读/日报批量缓存，每天1次，不实时调 |
| 黑天鹅/交易所跑路 | 现货、小资金、禁提现权限、不 all-in 单一交易所 |
| AI 直接下单/改参数出错 | 铁律：AI 只出信号和日报建议，下单和改参数由 Freqtrade/你把关 |
