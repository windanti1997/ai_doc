# 从人类记忆启发 Agent Memory 的开放性问题清单



## 使用目标



这份问题清单用于重新发散思考：



> 人类记忆中还有哪些机制，可以启发下一代 Agent Memory / Coding Agent 外置大脑 / Team Memory Infrastructure？



请不要急着给结论，每个问题都要深入思考一个个回答。 

请围绕这些开放问题继续扩展、类比、归纳、提出新机制、新架构、新评测、新产品方向。



---



# 一、记忆的本质：人类为什么需要记忆？



1. 人类记忆除了“保存过去”，还承担了哪些功能？

2. 人类记忆如何影响预测、决策、行动、情绪和身份认同？

3. 人类为什么不会把所有经历完整保存下来？

4. 人类如何判断哪些经历值得长期保留？

5. 人类记忆如何帮助一个人避免重复犯错？

6. 人类记忆如何帮助一个人在类似但不完全相同的场景中迁移经验？

7. 人类记忆如何在“准确保存”和“有用改写”之间取得平衡？

8. 人类记忆是否更像数据库、压缩算法、模拟器、行为调度器，还是身份建构系统？

9. 如果把记忆理解为“改变未来行为的系统”，会产生哪些 Agent Memory 设计方向？

10. 人类记忆中哪些机制最容易被今天的 Agent Memory 忽略？



---



# 二、注意与筛选：人类如何决定什么值得记？



1. 人类为什么会注意到某些信息，而忽略大量背景信息？

2. 目标、危险、奖励、异常、重复、情绪、社交反馈如何影响注意？

3. 人类如何在复杂环境中选择“值得编码”的信息？

4. 一次经历中，哪些细节会进入记忆，哪些会被丢弃？

5. 人类是否会根据当前身份、任务、情境选择不同的信息进入记忆？

6. 如果 Agent 也需要“记忆准入机制”，它应该参考人类注意系统的哪些特征？

7. Agent 在一次任务中产生大量工具日志、代码 diff、错误栈、对话内容时，如何决定哪些内容值得变成候选记忆？

8. 人类如何避免被无关信息淹没？

9. Agent Memory 如何避免“什么都记”导致的记忆污染？

10. 是否存在类似人类注意偏差的问题，会让 Agent 只记住某类显眼但不重要的信息？

# 三、情景记忆：人类如何保存一次具体经历？

6. 如果 Agent 记录一次任务经历，应该如何表示任务目标、执行动作、工具结果、失败现象和验证结果？

7. Agent 的 episodic memory 是否应该像人类经历一样具有边界、场景、参与者和结果？

8. 如何让 Agent 记住“不只是发生了什么”，还包括“为什么当时这么判断”？

9. 人类情景记忆中的细节会随时间模糊，Agent 是否也需要不同粒度的情景记忆？

10. 情景记忆如何成为后续语义知识和技能的原材料？
