# 命名与写作风格规范

## 总体风格

- 技术 + 认知 + 工程化结合
- 避免纯学术或纯产品宣传
- 以“机制解释”为主，而不是“工具说明”

## 标题风格

优先使用：

- 反常识句式
- 工程隐喻
- 行为驱动表达

示例：

- 别再把 Agent 记忆做成向量库了
- 给 Codex 装一个海马体
- 为什么你的 Agent 总像第一次见项目

## 关键词体系

- Memory Runtime
- Hook
- Checkpoint
- Skill Promotion
- Evidence-based Memory
- Memory Scope
- Memory Governance

## 写作原则

- 所有观点必须能映射到工程结构
- 所有概念必须可实现
- 所有隐喻必须能落到代码或架构
