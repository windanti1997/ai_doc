# 微信公众号发布文档：给 Agent 团队装一个共享大脑

## 系列定位

第一系列讲“一个 Agent 如何长经验”。

第二系列升级为团队视角：当 Agent 进入团队协作后，记忆不只是能力问题，而是知识治理问题。

统一系列名：

> 给 Agent 团队装一个共享大脑

统一核心判断：

> 团队记忆不是共享越多越好，而是要让正确经验在正确作用域内被正确使用。

---

## 01 标题

# 个人记忆不能直接变成团队规则

副标题：Personal、Project、Team、Organization，不同记忆不能混用。

## 公众号导语

如果一个 Agent 只服务你，记忆写错最多影响一次任务。但如果它进入团队，个人偏好、项目经验、团队规则、组织规范就不能混在一起。团队记忆的第一原则不是共享，而是分域。

## 关键小标题

1. 为什么共享记忆会带来风险
2. 个人记忆、项目记忆、团队记忆有什么区别
3. 个人经验什么时候可以升级成团队规则
4. 为什么每条 memory 都必须有 scope
5. 记忆不是越共享越好，而是要在正确作用域内生效

## 结尾引导

作用域解决的是“这条记忆属于谁”。但团队还要回答另一个问题：这条记忆凭什么可信？

---

## 02 标题

# Agent 记住什么不够，还要知道凭什么相信

副标题：Source、Evidence、Confidence，是团队记忆的可信底座。

## 公众号导语

同样一句“payment test 需要 Redis”，如果它来自模型猜测、用户口头提醒、测试结果、CI 日志、PR review，可信度完全不同。团队记忆必须知道来源，否则就会把猜测当事实。

## 关键小标题

1. 团队 memory 最大风险不是忘记，而是记错
2. 模型猜测、用户说明、工具验证不能等价
3. 每条记忆都应该带 source 和 evidence
4. confidence、last_verified、invalidated_by 如何工作
5. 没有来源的团队记忆，是组织级风险源

## 结尾引导

有了作用域和可信度，团队记忆还可以更进一步：不只是回忆过去，而是在未来正确时机主动提醒。

---

## 03 标题

# 最高级的记忆，是在未来正确时机自动提醒

副标题：Prospective Memory，让 Agent 记住“将来遇到 X 时要做 Y”。

## 公众号导语

很多工程事故不是因为团队不知道规则，而是关键时刻没人想起来。团队记忆如果只是被动检索，就还不够。更强的记忆应该是条件触发器：当修改 API、payment、auth、migration 时，自动提醒对应检查项。

## 关键小标题

1. 记忆不只回忆过去，也可以提醒未来
2. 什么是前瞻记忆
3. Coding Agent 里的触发条件：文件、模块、命令、风险级别
4. 从 memory item 到 trigger rule
5. 团队记忆的价值是关键时刻自动出现

## 结尾引导

当团队有多个 Agent 时，问题又升级了：是不是所有 Agent 都应该看到同一份记忆？

---

## 04 标题

# 团队不是共享所有知识，而是知道谁知道什么

副标题：从团队分工到 Multi-Agent Memory Routing。

## 公众号导语

真实团队不会让每个人掌握全部知识。高效团队知道谁懂支付、谁懂发布、谁能做安全 review。Agent Team 也一样。团队记忆的关键不是全量共享，而是路由。

## 关键小标题

1. 团队记忆不是大一统知识库
2. Planner、Coder、Reviewer、Security Agent 需要不同记忆
3. Memory Routing：按角色、任务、路径、风险召回
4. 谁知道什么、谁验证过什么、谁维护哪个 skill
5. 多 Agent 协作的核心是记忆边界清晰

## 结尾引导

一旦记忆可以共享和路由，就必须解决最后一个问题：错误记忆如何不进入团队系统？

---

## 05 标题

# Agent Memory 也需要免疫系统

副标题：如何防止错误经验、过期规则和坏 Skill 进入团队记忆。

## 公众号导语

团队记忆的价值很高，风险也很高。错误经验一旦进入共享 memory，可能被所有 Agent 召回；如果再升级成 skill 或 hook，错误会被自动化放大。所以团队 memory 必须有治理机制。

## 关键小标题

1. 共享记忆为什么需要审查
2. 候选记忆不能直接进入团队 memory
3. 审批、脱敏、权限、回滚、审计
4. 过期规则和冲突记忆如何处理
5. 团队 Agent 的共享大脑必须有治理层

## 结尾总结

第一系列解决单 Agent 如何长经验。第二系列解决团队如何安全共享经验。真正的 Agent Memory，不只是个人助手的长期记忆，而是团队工程经验的基础设施。
