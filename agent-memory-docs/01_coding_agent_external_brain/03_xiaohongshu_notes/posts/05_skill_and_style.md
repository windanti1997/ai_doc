### 02 先说结论

记录经历只是第一步。

真正的成长，是把经历变成稳定工作方式。

### 03 晋升链路

Episode -> Fact -> Rule -> Skill -> Automation

一次失败只是经历。

被验证后才可能变成知识。

### 04 什么是 Episode

Episode 是一次真实经历。

比如 payment webhook test 失败，错误是 Redis 连接不上。

它是经验的原材料。

### 05 什么是 Fact

Fact 是被验证的事实。

启动 Redis 后测试通过，所以可以记录：payment integration tests require Redis。

### 06 什么是 Skill

Skill 是可复用流程。

不是“知道 Redis”，而是“运行这类测试前如何检查依赖、启动服务、验证结果”。

### 07 什么值得升级

重复出现、流程稳定、结果可验证、错误成本高、未来复用概率高。

这样的经验才值得升级成 skill。

### 08 坏 Skill 很危险

错误经验如果变成 skill，会被反复执行。

如果再被 hook 自动触发，错误会被自动化放大。

所以需要 Curator。

### 09 总结

你给 Agent 什么记忆，它就会变成什么样的工程师。

Memory 的终点不是笔记，而是工作方式。

## 正文 Caption

Agent Memory 的终点不是 project_memory.md。

真正有价值的是：把经历升级成 skill。

一次失败只是 episode。

多次验证后，可能成为 fact。

作用域清晰后，可能成为 rule。

流程稳定后，才值得变成 skill。

再进一步，skill 可以变成 hook 或 automation。

但这件事必须谨慎。错误经验一旦变成 skill，影响会被放大。所以 Skill Promotion 前一定要检查证据、作用域、反例、验证结果和过期条件。

一句话：

真正强的 Agent，不是会写更多代码，而是会把经验变成稳定的工作方式。
