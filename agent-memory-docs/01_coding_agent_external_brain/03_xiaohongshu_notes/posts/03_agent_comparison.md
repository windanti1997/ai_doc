# 小红书 03：我对比了十几个 Coding Agent，发现大多数还不会长经验

## 标题

我对比了十几个 Coding Agent，发现大多数还不会长经验

## 封面文案

大多数 Coding Agent 还不会长经验

副标题：有 memory，不等于会成长

## 9 张卡片文案

### 01 封面

我对比了十几个 Coding Agent

发现大多数还不会长经验。

### 02 先说结论

现在很多工具都说自己有 memory。

但“会记录”和“会成长”不是一回事。

真正的分水岭是 Skill Promotion。

### 03 L1 规则型

规则型 Agent 会读 AGENTS.md、CLAUDE.md、rules。

像新人看手册。

优点是可控，缺点是不会自己从失败中学习。

### 04 L2 上下文型

上下文型 Agent 当前任务很强。

能读 workspace、跟 IDE、理解文件。

但 session 后，很多经验不会沉淀。

### 05 L3 自动记忆型

自动记忆型会总结偏好、习惯、测试命令。

但问题是：为什么记？证据是什么？能不能修正？

自动记忆不等于正确记忆。

### 06 L4 生命周期型

生命周期型开始有 hooks、skills、commands、compaction。

记忆不只是结束后总结，而是在 Agent 工作过程中发生。

### 07 L5 进化型

进化型关注 Dream、Distill、Curator、Skill Learning。

它们开始把经历整理成知识，把知识提炼成技能。

### 08 谁更值得关注

Codex、Claude Code、OpenCode、Kiro、Qwen Code 正在进入 L4。

Hermes、MiMo Code 更接近 L5。

### 09 总结

真正的 Agent Memory，不是有没有记录。

而是能不能把一次成功经验，变成以后自动执行的能力。

## 正文 Caption

我觉得评价 Coding Agent 的 memory，不能只看它有没有“记住东西”。

更关键的是：它能不能从经历里长出技能。

可以粗略分五层：

L1：能读规则。

L2：能维持上下文。

L3：能自动记忆。

L4：能生命周期治理。

L5：能技能进化。

大多数工具还在 L1-L3。Codex、Claude Code、OpenCode、Kiro、Qwen Code 已经有进入 L4 的入口。Hermes、MiMo Code 最接近 L5。

真正的分水岭不是有没有 memory，而是有没有 Skill Promotion。
