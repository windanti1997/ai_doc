# 小红书 01：别再把 Agent 记忆做成向量库了

## 标题

别再把 Agent 记忆做成向量库了

## 9 张卡片文案

### 01 封面

别再把 Agent 记忆做成向量库了

RAG 是查资料，Memory 是长经验。

### 02 痛点

很多 Coding Agent 不是不会写代码。

它的问题是：刚踩过的坑，下次还会踩；刚确认过的测试命令，下次又忘。

这不是缺上下文，而是缺经验系统。

### 03 反常识

人类记忆不是硬盘。

它不是为了完整保存过去，而是为了让过去改变下一次行动。

Agent Memory 也应该这样。

### 04 RAG vs Memory

RAG 的动作是检索。

Memory 的动作是改变行为。

一个负责“查到什么”，一个负责“下次怎么做”。

### 05 什么值得记

不是所有日志都值得存。

更值得进入长期记忆的是：用户纠正、失败测试、关键错误栈、高风险模块、重复流程。

### 06 记忆要有证据

好的 memory 不能只写一句结论。

它要知道：来自哪个命令、哪个文件、哪个错误、最后怎么验证。

没有证据的记忆，只是模型的自我感觉。

### 07 记忆也要遗忘

不会遗忘的 Agent，会被旧经验污染。

测试命令会变，架构会变，workaround 会过期。

Memory 必须能降权、失效、归档。

### 08 最高级的记忆

最高级的 memory 不是 project_memory.md。

而是把重复经验升级成 workflow、skill、hook 和 automation。

### 09 总结

Agent Memory 的目标不是记住更多。

而是让 Agent 下一次做得更好。

## 正文 Caption

如果你的 Coding Agent 每次都像第一次见这个项目，问题可能不是模型不够强，而是它没有真正的记忆系统。

很多人一提 Agent Memory，就想到聊天记录、摘要、向量库、RAG。它们当然有用，但更像资料库，不是经验系统。

真正有价值的记忆，应该能筛选、带证据、会遗忘、能巩固，最后升级成 workflow 或 skill。

一句话：

RAG 是查资料，Memory 是长经验。

后面我会继续拆：为什么 Coding Agent 总像第一次见项目，以及 Hook Memory Runtime 怎么做。
