## 9 张卡片文案

### 01 封面

为什么你的 Agent 总像第一次见这个项目？

长上下文不等于长期记忆。

### 02 熟悉的崩溃

你说过用 pnpm，它下次还是 npm。

你说过别改 generated 文件，它下次又改。

你说过先跑测试，它还是直接总结完成。

### 03 问题在哪

它不是没看到信息。

而是没有把信息沉淀成经验。

上下文是“这次看见什么”，记忆是“下次怎么做得更好”。

### 04 四层记忆

Coding Agent 至少需要四层记忆：

Working、Episodic、Semantic、Procedural。

少一层都会断。

### 05 Working Memory

Working Memory 负责当前任务不断线。

目标、约束、todo、修改文件、测试结果、下一步计划，都属于这一层。

### 06 Episodic Memory

Episodic Memory 记录真实经历。

命令、错误栈、diff、失败尝试、用户纠正、最终验证结果。

它是经验的原材料。

### 07 Semantic Memory

Semantic Memory 沉淀项目知识。

例如：项目用 pnpm；generated 文件不要手改；payment test 需要 Redis。

但每条都要有证据和作用域。

### 08 Procedural Memory

Procedural Memory 是可复用技能。

不是“知道什么”，而是“怎么做”。

例如 ci-debug.skill、api-change.workflow。

### 09 总结

Agent Memory 不是更长上下文。

而是 Write、Retrieve、Consolidate、Forget、Promote 的完整生命周期。

## 正文 Caption

很多 Coding Agent 的问题不是“看得不够多”，而是“不会沉淀经验”。

长上下文只能帮它在这一次任务里保留更多信息，但任务结束后，如果没有结构化 memory，它下次还是会像新人一样重新摸索。

我觉得 Coding Agent 至少要有四层记忆：

Working：当前任务状态。

Episodic：真实执行轨迹。

Semantic：项目知识。

Procedural：可复用流程和 skill。

再加上五个动作：写入、召回、巩固、遗忘、晋升。

这才是 Agent 从“会执行”走向“会长经验”的基础。
