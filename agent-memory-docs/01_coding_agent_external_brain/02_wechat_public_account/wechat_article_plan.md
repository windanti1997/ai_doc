## 01 标题

# 别再把 Agent 记忆做成向量库了

副标题：人类记忆不是硬盘，Agent Memory 也不该只是 RAG。

## 公众号导语

这篇不讲工具，也不讲模型参数。我们先问一个更底层的问题：人类为什么能“长经验”？如果记忆只是保存过去，那向量库就够了。但人类记忆真正厉害的地方，是能筛选、加权、巩固、遗忘，并把经历变成下一次行动。

## 关键小标题

1. Agent 不是缺上下文，而是缺经验系统
2. RAG 是查资料，Memory 是长经验
3. 睡眠、情绪、遗忘、习惯，其实都属于记忆生命周期
4. Agent Memory 的第一步不是存储，而是筛选
5. Agent 的长期记忆，会塑造它的工作风格

## 结尾引导

如果记忆不是存储，那 Coding Agent 就不能只靠 RAG。下一篇继续讲：为什么你的 Agent 总像第一次见这个项目？

## 02 标题

# 为什么你的 Agent 总像第一次见这个项目？

副标题：Coding Agent 需要的不是更长上下文，而是记忆生命周期。

## 公众号导语

你明明告诉过它这个项目用 pnpm，不要用 npm；你明明让它跑过测试；你明明纠正过它不要改 generated 文件。但下一次，它还是忘了。问题不是它没有上下文，而是没有把上下文沉淀成经验。

## 关键小标题

1. 长上下文不是长期记忆
2. Coding Agent 至少需要四层记忆
3. 记忆必须带证据、作用域和状态
4. 没有遗忘机制的记忆，迟早会污染任务
5. Promote：从经历升级成技能

## 结尾引导

有了这套框架，我们再看今天的 Codex、Claude Code、OpenCode、Hermes、MiMo，就会发现：大多数还只是“记录”，还没有真正“长经验”。

## 03 标题

# 我对比了十几个 Coding Agent，发现大多数还不会长经验

副标题：Codex、Claude Code、OpenCode、Hermes、MiMo 谁真正开始做记忆系统？

## 公众号导语

现在几乎所有 Coding Agent 都在说自己有上下文、有记忆、有规则。但它们做的是同一件事吗？不是。有些只是读规则，有些只是维护上下文，有些会自动总结，少数开始把经历变成技能。

## 关键小标题

1. 五类 Coding Agent 记忆系统
2. 规则型：会看手册，但不会长经验
3. 上下文型：短期记忆好，但不一定成长
4. 自动记忆型：会做笔记，但不一定可治理
5. 生命周期型：Hook 是关键分水岭
6. 进化型：Hermes / MiMo 为什么值得关注

## 结尾引导

现有工具还不完美，但 Codex、Claude Code、OpenCode 已经给了我们足够入口。下一篇就讲：如何自己给 Coding Agent 装一个记忆 Runtime。

## 04 标题

# 给 Codex 装一个海马体

副标题：用 Hook、Checkpoint 和 Verifier 做 Agent Memory Runtime。

## 公众号导语

我们不一定要训练新模型，才能让 Agent 有更好的记忆。更现实的方法，是把 Agent 的关键生命周期事件接出来：任务开始、用户输入、工具调用、上下文压缩、任务停止。这些都是记忆写入点。

## 关键小标题

1. Hook 为什么比事后总结更可靠
2. SessionStart：启动时加载最小必要记忆
3. PostToolUse：工具结果是最可靠的证据
4. PreCompact：长任务压缩前必须写 checkpoint
5. Stop：防止 Agent 假完成
6. Dream / Distill / Curator：离线巩固和技能化

## 结尾引导

到这里，Agent 已经可以记录经历。但真正的成长，还要把经历变成技能。下一篇讲：从踩坑到 Skill。

## 05 标题

# 从踩坑到技能：让 Agent 不再重复犯错

副标题：经验如何升级成 Skill，并塑造 Agent 的工作风格。

## 公众号导语

一个工程师真正有经验，不是因为他记住了更多日志，而是因为他形成了稳定工作方式。Agent 也一样。Memory 的终点不是 project_memory.md，而是 skill、workflow、hook、command 和 subagent playbook。

## 关键小标题

1. 只会记笔记的 Agent 仍然像新人
2. Episode → Fact → Rule → Skill → Automation
3. 什么经验值得升级成 Skill
4. 错误经验也可能变成坏技能
5. 记忆如何塑造 Agent 的工作风格
6. 给 Agent 什么记忆，它就会变成什么样的工程师

## 结尾引导

第一系列到这里完成闭环：单个 Agent 如何从经历中成长。后续可以继续讲团队版：个人经验如何升级成团队记忆，错误记忆如何防止污染组织。
