# 别再把 Agent 记忆做成向量库了

> 人类记忆不是硬盘，Agent Memory 也不该只是 RAG。

如果一个工程师每天都在重复踩同一个坑，你会觉得他聪明吗？

现在很多 Coding Agent 就像这样的工程师。

它能写代码，能改文件，能跑测试，能读日志，能调用工具，甚至能连续执行几十步任务。

但它仍然经常出现一种很“新人”的表现：

- 刚踩过的坑，下次还会踩；
- 刚确认过的测试命令，下次又忘；
- 刚理解的项目约束，过几轮就丢；
- 刚失败过的方案，换个任务又重新尝试；
- 长任务压缩之后，目标、上下文、用户要求都被冲淡。

这说明一个问题：

> Agent 不只是缺上下文，而是缺经验系统。

很多人一说 Agent Memory，第一反应是存聊天记录、存摘要、建向量库、下次检索出来。

这当然有用。

但这还不是完整的记忆。

它更像资料库，或者搜索系统。

真正值得学习的人类记忆，不是“大脑能存多少东西”，而是：

> 大脑如何让过去的经历影响未来的行为。

人类记忆不是硬盘。

它不是为了完整保存过去，而是为了预测未来、指导行动、形成习惯、塑造判断，甚至影响一个人的性格。

所以，Agent Memory 也不应该只回答：

> 过去发生过什么？

它更应该回答：

> 什么值得记？这条记忆有多重要？它现在还成立吗？它会如何影响下一次行动？它能不能变成技能？

这才是 Agent Memory 真正值得做的方向。

## 一、记忆不是为了保存过去，而是为了指导未来

很多人把记忆理解成“存储”。

比如：把聊天记录存起来，把代码片段存起来，把项目文档存起来，把摘要写进 memory，把内容向量化后检索。

这种思路更像数据库。

数据库关心的是：信息有没有保存？能不能查到？查得准不准？

但人类记忆关心的不是这些。

人类记忆更关心：这件事以后还会不会遇到？它对我重要吗？它带来了收益还是惩罚？下次遇到类似情况，我应该怎么反应？这次经历是否改变了我的判断？

所以，人类记忆的核心不是保存，而是影响未来行为。

一个工程师真正“有经验”，不是因为他能背出所有历史日志，而是因为他下次遇到类似问题时：

- 更快识别风险；
- 更快定位根因；
- 避免重复踩坑；
- 选择更稳妥的方案；
- 形成可复用的排查流程。

对应到 Agent，就是：

> Agent Memory 的目标不是让 Agent 记住更多，而是让 Agent 行动得更好。

这也是它和 RAG 的根本区别。

RAG 更像“查资料”。

Memory 更像“长经验”。

## 二、记忆相关行为属于同一条生命周期

睡眠、情绪、遗忘、练习、习惯、性格，这些看起来像零散话题。

但如果从记忆系统看，它们其实都是同一条生命周期里的不同环节。

可以把人类记忆相关行为整理成这样一条链路：

```text
注意
  ↓
编码
  ↓
重要性加权
  ↓
重复与练习
  ↓
巩固
  ↓
召回
  ↓
再巩固
  ↓
遗忘
  ↓
习惯化
  ↓
自我叙事与性格
```

这条链路说明：

- 注意决定什么信息会被看到；
- 编码决定什么信息会被写入；
- 情绪、奖励、惩罚、压力决定什么信息更重要；
- 重复和练习让记忆变强；
- 睡眠和休息帮助记忆巩固；
- 召回让记忆参与当前决策；
- 再巩固让旧记忆被新经验修改；
- 遗忘让系统摆脱无用和过期信息；
- 习惯化让经验变成自动行为；
- 长期记忆和自我叙事会影响一个人的行为风格。

所以，这些不是散点。

它们都服务于同一个目标：

> 让重要经验影响未来行为。

## 三、Agent Memory 的第一步不是存储，而是筛选

人每天接收的信息非常多。

但真正进入长期记忆的很少。

你不会记住今天刷过的每一条信息，也不会记住路上每个行人的衣服颜色。

大脑首先会做一件事：筛选。

Agent 也一样。

一次任务会产生大量内容：用户 prompt、代码 diff、shell 输出、测试日志、错误栈、中间猜测、临时计划、工具调用结果。

这些都可以存。

但不应该都进入长期记忆。

如果什么都记，memory 很快就会变成垃圾堆。

所以 Agent Memory 的第一层应该不是 storage，而是 gate。

也就是：

```text
Memory Gate：判断什么值得进入记忆
```

它需要判断：

- 是否影响未来任务？
- 是否被测试验证？
- 是否重复出现？
- 是否来自用户明确纠正？
- 是否和高风险模块有关？
- 是否代表某个失败根因？
- 是否值得长期保留？

普通日志不一定值得记。

但反复出现的测试失败、用户明确强调的偏好、被验证过的项目规则、线上事故根因，就值得进入长期记忆。

> 记忆的第一步不是写入，而是选择。

## 四、好的记忆不是一句摘要，而是带证据的经验单元

很多 Agent 记忆系统喜欢写一句摘要：

```text
这个项目测试需要 Redis。
```

这句话有用，但不够。

因为它缺少上下文。

它没有说明：哪个项目？哪类测试？哪次失败发现的？相关文件是什么？启动 Redis 后是否真的通过？现在是否还成立？

更好的记忆应该像这样：

```text
任务：修复 payment webhook integration test
命令：pnpm test payment-webhook.test.ts
现象：Redis connection refused
根因：本地 Redis 未启动
验证：启动 Redis 后测试通过
相关文件：docker-compose.yml、tests/payment-webhook.test.ts
可复用经验：运行该类测试前需要检查 Redis
```

这才是一个可用的记忆单元。

它不仅能被检索，还能被验证、更新、失效和升级。

> 好的 Agent Memory 不是一句摘要，而是带证据、作用域和结果的经验单元。

## 五、Agent 没有情绪，但记忆需要重要性权重

人类不是平等地记住所有经历。

一次严重事故，一次关键失败，一次用户强烈纠正，往往会被记很久。

情绪给 Agent 的启发，不是让 Agent 拥有情绪。

而是让 Agent Memory 拥有重要性评分。

比如：

```text
用户明确纠正：高权重
测试失败根因：高权重
线上事故：极高权重
安全风险：极高权重
普通 lint warning：低权重
重复出现三次的问题：高权重
```

这样，Agent 不会把所有记忆一视同仁。

它会更重视高风险、高频重复、被用户纠正、被测试验证、影响范围大的信息。

对应字段可以是：

```text
importance_score
risk_score
failure_frequency
user_correction_weight
business_impact
last_verified
```

> Agent 没有情绪，但 Agent Memory 需要重要性权重。

## 六、睡眠和做梦的工程启发：离线巩固

睡眠、做梦、休息，在记忆系统里属于“巩固”。

人类白天经历大量事情，睡眠和休息期间，大脑会对部分记忆进行再激活、整合、压缩和稳定化。

重要经历不会总是在发生当下就变成长期知识。

很多时候，它需要离线整理。

这对 Agent 非常关键。

主 Agent 在执行任务时，已经要读代码、改文件、跑测试、解释错误、做计划、调工具、回应用户。

让它同时维护长期记忆，容易导致两件事都做不好。

更合理的是角色分离：

```text
主 Agent：负责当前任务执行
Memory Writer：记录关键经历
Consolidator / Dream：定期合并、去重、压缩
Distiller：把重复流程提炼成 skill
Curator：归档过期或低价值记忆
```

这就是“睡眠”的工程启发：

```text
白天执行任务 → 产生经历 → 离线整理 → 沉淀知识 → 形成技能
```

> Agent Memory 需要离线巩固机制，不能只靠在线上下文堆叠。

## 七、遗忘不是缺陷，而是长期可用性的前提

人会遗忘。

这不是纯粹缺陷。

在很多情况下，遗忘是一种能力。

因为如果所有细节都永久保留，人会被无关信息淹没。

Agent 也是一样。

如果 memory 只增不减，最后会变成污染源。

Coding Agent 尤其需要遗忘，因为代码库会变：

```text
npm 迁移到 pnpm
Redis 依赖被 mock 替代
API v1 变成 v2
旧目录被重构
旧 workaround 被删除
测试命令发生变化
```

过去正确的经验，未来可能变成错误的指导。

所以每条长期记忆都应该有状态：

```text
confidence
last_verified
expires
stale
archived
invalidated_by
superseded_by
```

遗忘不一定是删除。

也可以是降权、归档、标记过期、等待重新验证、被新规则覆盖。

> 不会遗忘的 Agent，会被旧经验误导。

## 八、记忆最终会变成自动行为

人类最强的记忆，不一定是能说出来的知识。

而是能自然做出来的动作。

熟练工程师修 bug，不会每次都从零开始想。

他会形成固定流程：

```text
复现问题
定位边界
写失败测试
最小修改
跑验证
检查 diff
提交 PR
```

这就是程序记忆。

Agent 也应该这样。

最高级的 Agent Memory 不是 project_memory.md，而是：

```text
tdd-loop.skill
ci-debug.skill
api-change.workflow
migration-checklist
release-command
security-review-agent
```

也就是：

```text
经验 → 规则 → 流程 → 技能 → 自动化
```

> Agent Memory 的终点不是长期笔记，而是可复用技能。

## 结语

人类记忆不是硬盘。

它不是为了完整保存过去，而是为了让重要经验影响未来行为。

所以 Agent Memory 也不应该只回答“过去发生过什么”。

它应该回答：

```text
什么值得记？
这条记忆为什么重要？
它适用于哪里？
它现在还成立吗？
它应该如何影响下一次行动？
它能不能变成技能？
```

最终一句话：

> Agent Memory 的目标不是让 Agent 记住更多，而是让 Agent 形成经验、习惯和风格。
