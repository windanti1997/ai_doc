# 执行复盘：为什么上一版 PR 会被简化、改写、跑偏

## 1. 这次错误是什么

上一版 PR 犯了三个严重问题：

1. `archive_published/` 里的文章不是之前对话确认过的最后一版，而是我根据记忆和摘要重新压缩/改写的版本；
2. `contracts/` 只覆盖了 01–06，且后续契约没有按“每篇单独、足够详细、含封面提示词”的要求完整展开；
3. `skills/` 里的 skill 不是当前对话里反复迭代后的完整版本，而是被我重新概括成了短版。

这不是“内容还可以再补充”的问题，而是执行方法错了：我把“整理资料”做成了“重新生成资料”。

## 2. 直接原因

### 2.1 把归档误当成摘要

用户要求的是“已发布文章最后一版归档”，我却把它处理成“给其他 Agent 看的文章摘要/重写版”。

归档目录的定义应该是：

> 只存最终原文，不改写、不压缩、不润色、不补写。

只要不能确认是最终原文，就不能放进归档并伪装成最终版。

### 2.2 把契约误当成大纲

用户要求契约“其他 Agent 读了就知道怎么写”，我上一版每篇只写了标题、边界、场景和封面，缺少：读者情绪、叙事路线、必写/禁写、段落推进、跑偏风险、首屏要求、结尾要求、评审重点等。

契约不是大纲。契约是防跑偏的执行协议。

### 2.3 把 skill 误当成规则摘要

当前写作 skill 是经过多轮失败样本修出来的，尤其包括：标题无上下文测试、全评审严格模式、反评审讨好、禁止自证、禁止金句三连、公众号裁剪安全区等。

我上一版只保留了少量原则，丢掉了真正能防止其他 Agent 犯错的硬规则。

## 3. 为什么之前的约束没有阻止这个错误

### 3.1 验收标准没有被我转成逐项 checklist

用户已经给了 5 条验收标准，但我没有在生成前逐条建立“文件级验收清单”。结果我只满足了目录形状，没有满足内容深度。

### 3.2 “不要偷懒”没有变成可执行门槛

我没有规定每个契约必须包含哪些章节、最低信息量、是否足以让另一个 Agent 独立写作。这导致契约看似完整，实际只是简化提纲。

### 3.3 没有区分三类文件的事实属性

- 归档：事实源，只能复制最终原文；
- 契约：执行约束，可以创作和扩展；
- skill：流程规则，必须保留完整硬规则。

我把三类文件都当成“可整理摘要”，导致最关键的归档被污染。

### 3.4 没有执行 source-of-truth 检查

当我找不到文章 01/02 的完整最终原文时，正确动作应该是：标记缺源、停止伪装、说明需要从公众号后台或原对话导出回填。

错误动作是：凭摘要重写一版放进归档。

## 4. 修正后的执行方法

以后做这类资料包，必须先跑以下流程：

```text
1. 先分类文件属性：归档 / 契约 / skill / workflow / 参考资料。
2. 对归档类文件执行 source-of-truth 检查：没有最终原文就标记缺源，不能重写。
3. 对契约类文件执行完整性检查：每篇必须独立、详细、含封面提示词。
4. 对 skill 类文件执行版本完整性检查：不能用摘要替代当前工作版。
5. 生成后按用户验收标准逐条核对。
```

## 5. 新增硬规则

### 5.1 归档禁止改写

归档目录只能保存最终原文。禁止摘要、压缩、润色、重写、补写。如果最终原文不可用，文件必须标记 `NEEDS_SOURCE_BACKFILL`，不能假装完成。

### 5.2 契约不能低于可执行粒度

每篇契约至少包含：系列位置、唯一问题、读者心声、认知推进、必写场景、禁写内容、叙事路线、开头要求、中段推进、结尾任务、封面提示词、跑偏风险、评审重点。

### 5.3 skill 不能摘要化

skill 文件必须保留硬规则、反例、评审输出格式、失败优先流程。短摘要只能作为 README，不能替代 skill 本体。

### 5.4 PR 自审必须逐项引用验收标准

提交前必须把用户验收标准逐条转成自审表。如果某项无法完全满足，必须明确标注，而不是用模糊语言掩盖。
