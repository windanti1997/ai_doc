# 《给 AI / Coding Agent 装一个外置大脑》Series Bible

## 1. 系列一句话定位

这个系列不是泛泛讲 AI 技巧，也不是提示词教程，而是围绕一个核心问题：

> AI / Agent 为什么不能像真正的助手一样，从过去的纠正、失败和经验里持续变好？

系列要持续回答：

> 怎么给 AI / Coding Agent 装一个真正有用的外置大脑。

## 2. 核心读者

主要读者不是 AI 研究员，而是：经常用 ChatGPT / Claude / Cursor / Codex / Claude Code 的普通用户；想把 AI 用成长期生产力的人；尝试 Agent、自动化、AI 副业、AI 产品的技术人；被 AI 反复忘记、误会、乱记、重复犯错折磨过的人。

文章必须先让普通 AI 用户共鸣，再逐步看到背后的 Agent Memory 设计问题。

## 3. 系列底层公式

> Agent Memory = Experience × Trigger × Verification × Lifecycle  
> AI 记忆 = 经历 × 触发 × 验证 × 生命周期

不要每篇都直接讲公式。每篇只讲其中一个局部，不能一篇把完整体系讲完。

## 4. 第一阶段：普通用户痛点六连

### 01 忘了：纠正没有变成提醒

标题方向：为什么 AI 明明被你纠正过，下次还是犯同样的错？

核心情绪：我明明说过了，你怎么又忘了？

只讲：纠正没有沉淀成下一次行动前的可用经验。

### 02 误会了：记住原话但没懂意思

标题方向：AI 最烦人的不是忘了你，而是误会你。

核心情绪：不是这个意思啊。

只讲：AI 找到了旧话，但没理解当时语境，导致旧话被用错。

### 03 乱记了：临时要求变长期偏好

标题方向：AI 记住你以后，怎么反而更难用了？

核心情绪：我上次只是那次需要，不是每次都要这样。

只讲：一次要求被固化成默认设置，导致越记越窄。

### 04 想不起来：记忆有了但没触发

标题方向：AI 明明记住了，为什么该用的时候还是想不起来？

核心情绪：你不是知道吗？那刚才为什么不用？

只讲：任务开始前没有触发相关记忆。

### 05 记得对不对：经验必须验证

标题方向：AI 的记忆也会骗人：它怎么知道哪次经验是对的？

核心情绪：你别把一次失败经验也当成规则啊。

只讲：触发出来的经验是否可靠，必须验证。

### 06 什么时候该忘：记忆生命周期

标题方向：AI 不该什么都记住：有些记忆，过期了就该删。

核心情绪：那是以前的我，不是现在的我。

只讲：过期、冲突、低质量、场景失效的记忆需要降权、更新或淘汰。

## 5. 第二阶段：外置大脑设计六连

### 07 经历怎么变经验：Experience Capture

标题方向：AI 不是缺记性，是缺一套把经历变经验的系统。

核心情绪：你不能每次做完就算了，至少得知道这次学到了什么。

只讲：如何从对话、修改、失败、用户纠正中提取 memory candidate。

### 08 经验放在哪里：Memory Layers

标题方向：AI 的记忆不能全塞一起：有些属于我，有些属于项目，有些属于团队。

核心情绪：别把我的个人偏好、项目规则、团队规范混在一起。

只讲：个人记忆、项目记忆、团队记忆、任务临时记忆的分层。

### 09 多个 Agent 怎么共享经验：Team Memory

标题方向：一个 Agent 踩过的坑，为什么另一个 Agent 还要再踩一遍？

核心情绪：这不是刚有人修过吗，怎么换个 Agent 又重来？

只讲：团队记忆、跨 Agent 经验复用、共享经验边界。

### 10 怎么防止坏记忆污染系统：Memory Immune System

标题方向：给 AI 装记忆之前，先想清楚怎么防止它被坏记忆带偏。

核心情绪：记忆多了不可怕，可怕的是错的也被当成真的。

只讲：记忆污染、冲突、恶意注入、低质量经验的免疫机制。

### 11 经验怎么升级成技能：Distill / Skill Promotion

标题方向：AI 真正变强，不是记住更多，而是把反复成功的经验变成技能。

核心情绪：同一类事做了这么多次，为什么还不能形成套路？

只讲：从候选经验到稳定 skill 的提炼、晋升、验收。

### 12 最终收束：外置大脑不是资料库，而是成长系统

标题方向：给 AI 装外置大脑，最后不是为了让它记得更多。

核心情绪：我想要的不是会背聊天记录的 AI，而是能越用越顺手的助手。

只讲：回收全系列，把 Experience / Trigger / Verification / Lifecycle 串成一个成长系统。

## 6. 系列统一写法

每篇必须从真实使用场景进入；只推进一个核心问题；先让普通用户共鸣，再轻轻点机制；结尾引出下一篇但不展开；保持“给 AI 装一个外置大脑”的系列感。

## 7. 禁止事项

禁止写成 AI 记忆百科、提示词教程、产品需求文档、纯技术架构介绍、作者自嗨式思考。禁止每篇重复总论。禁止一篇把 Experience / Trigger / Verification / Lifecycle 全讲完。

## 8. 稳定表达

可重复但不能滥用：给 AI 装一个外置大脑；记忆不是存档，而是下一次行动前的判断；好的 AI 记忆不是记得多，而是用得准；AI 要记住过去，也要知道过去什么时候不该继续用。
