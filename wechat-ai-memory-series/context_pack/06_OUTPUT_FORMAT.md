# 最终输出格式

每次让其他 Agent 写这个系列文章时，必须按以下顺序输出。

## 1. Series Alignment Card

```markdown
# Series Alignment Card

## 本篇编号
第 X 篇

## 前一篇讲什么
上一篇只讲：{{上一章边界}}

## 本篇只讲什么
这一篇只讲：{{本篇边界}}

## 下一篇会讲什么
下一篇会讲：{{下一章边界}}

## 本篇不能碰的内容
- 不讲 {{禁止主题 1}}
- 不讲 {{禁止主题 2}}
- 不讲 {{禁止主题 3}}

## 本篇读者心声
{{读者最想说的话}}

## 本篇主场景
{{具体场景}}

## 跑偏风险
最容易跑偏成：
1. {{风险 1}}
2. {{风险 2}}
3. {{风险 3}}

## 自检结论
如果正文偏离以上边界，必须重写。
```

## 2. 写前卡片

```markdown
# 写前卡片

【本篇标题】
【本篇读者是谁】
【本篇独有情绪】
【读者内心台词】
【本篇认知推进】
【本篇叙事形状】
【本篇主场景】
【本篇主比喻】
【情绪递进】
【禁用内容】
【失败判定】
```

## 3. 正文

输出 doocs / 公众号可复制 Markdown 正文。正文要求：用 `#` 标题、自然 `##` 小标题、引用承载对话或关键句、重点可加粗、不用代码块包装正文金句、普通段落 2～4 句、不强行编号、不机械逐句断行。

## 4. 发布配套

```markdown
## 发布配套

**公众号标题**
> {{主标题}}

**备选标题**
> {{备选 1}}
> {{备选 2}}

**摘要**
> {{摘要}}

**封面大字**
> {{封面大字}}

**封面副标题**
> {{封面副标题}}

**封面提示词**
{{完整封面提示词}}
```

## 5. 严格复审

必须用 15 个评审角色，每个角色按以下格式：

```markdown
**1. 标题评审 Subagent**
【结论】通过 / 必须改 / 不适用
【最可能失败点】...
【红线检查】...
【证据】...
【修改方向】...
```

如果任一硬门槛为“必须改”，不允许交付最终稿，必须先修改再复审。
