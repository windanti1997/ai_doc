# Article Contract 10：给 AI 装记忆之前，先想清楚怎么防止它被坏记忆带偏

## 文章编号

给 AI 装一个外置大脑 10

## 系列位置

第 9 篇讲团队经验共享。本篇讲共享和长期记忆带来的风险：坏记忆、冲突记忆、恶意注入、低质量经验一旦进入系统，会让 Agent 越学越偏。

## 标题方向

主标题：给 AI 装记忆之前，先想清楚怎么防止它被坏记忆带偏

备选标题：AI 记忆多了不可怕，可怕的是错的也被当成真的 / 外置大脑也需要免疫系统

## 本篇只解决一个问题

Memory Immune System：如何防止坏记忆污染个人、项目、团队记忆。

## 读者核心情绪

> 这条经验到底能不能信？别把错的也传给所有 Agent。

## 本篇认知推进

上一篇讲共享团队经验。本篇提醒：共享之前必须先做免疫。下一篇会讲经过验证且反复成功的经验，如何晋升为 skill。

## 必写场景

至少写 3 个：

1. 一个临时 workaround 被写进团队记忆，后来所有 Agent 都复用它；
2. 用户一次情绪化评价被 AI 当成长期规则；
3. 某个 Agent 生成了错误总结，另一个 Agent 又拿它当事实；
4. 项目规则更新后旧规则与新规则冲突，AI 不知道听谁的。

## 禁止展开

不讲安全攻防大科普；不讲 Prompt Injection 全教程；不讲权限系统细节；不讲数据库实现。本文只讲坏记忆进入系统后为什么危险，以及需要怎样的免疫思路。

## 推荐开头

从一个“错误经验被传播”的场景开始：某个 Agent 为了临时过测试改了一个 hack，后来它被记录成经验，另一个 Agent 继续复用，问题从一个局部 bug 变成团队习惯。

## 叙事路线

场景：坏经验被复用。冲突：记忆让错误传播更快。推进：外置大脑不是越开放越好。机制：来源、验证、置信度、冲突检测、过期检查、人工确认。落点：没有免疫系统的记忆，会把 Agent 训练成更自信的错误助手。

## 结尾任务

落到：记忆系统真正难的不是存进去，而是决定哪些东西不能进去，哪些进去后也不能随便信。引出第 11 篇：经过免疫和验证的经验，如何升级成稳定 skill。

## 封面提示词

封面大字：坏记忆 / 会带偏 AI。封面副标题：外置大脑也需要免疫系统。画面：轻漫画、浅底、黑红大字、黄色强调条。AI 正把一张写着“临时 hack”的红色便签塞进大脑仓库，风逆紧急拦住；旁边有过滤网、警报灯、病毒图标但不要恐怖。黄色便签：“没验证”“会污染团队”“先拦住”。中央安全区兼容公众号裁剪。

## 跑偏风险

不要写成纯安全文章。必须与本系列的记忆质量、经验复用、Agent 成长主线一致。

## 评审重点

读者能不能理解：记忆系统越强，错误记忆的破坏力也越强。
