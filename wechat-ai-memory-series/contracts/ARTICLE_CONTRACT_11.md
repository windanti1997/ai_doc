# Article Contract 11：AI 真正变强，不是记住更多，而是把经验变成技能

## 文章编号

给 AI 装一个外置大脑 11

## 系列位置

第 10 篇讲坏记忆免疫。本篇讲通过验证、反复成功、适用范围清晰的经验，如何从“记忆”晋升成“技能”。

## 标题方向

主标题：AI 真正变强，不是记住更多，而是把经验变成技能

备选标题：同一类事做了这么多次，AI 为什么还不能形成套路？ / 从记忆到技能：让 Agent 不只是会回忆，而是会成长

## 本篇只解决一个问题

Distill / Skill Promotion：哪些经验可以升级成 skill，升级后如何被调用、复用和验收。

## 读者核心情绪

> 同一类事做了这么多次，为什么还不能形成套路？

## 本篇认知推进

上一篇讲坏记忆不能随便进系统。本篇讲好经验不能永远停留在碎片记忆里。下一篇会全系列收束：外置大脑不是资料库，而是成长系统。

## 必写场景

至少写 3 个：

1. 风逆反复要求 AI 写文章前先做写前卡片、再评审，后来这应该变成固定写作 skill；
2. Coding Agent 多次修同类 bug 后，应形成“先查配置、再看日志、最后改代码”的调试流程；
3. 多次 PR review 后形成“先跑测试、再检查 diff、再总结风险”的执行套路；
4. 经验还没稳定时只能做记忆，稳定后才适合晋升成 skill。

## 禁止展开

不讲技能市场；不讲插件系统实现；不讲具体模型微调；不讲完整产品路线图。本文只讲经验如何晋升为技能。

## 推荐开头

从当前写作 skill 的形成过程进入：一开始只是一次次被用户挑刺，后来这些失败样本沉淀成标题评审、反评审讨好、禁止自证、封面裁剪等规则。它已经不只是记忆，而是一套写作技能。

## 叙事路线

场景：同类任务反复发生。冲突：每次都重新提醒太浪费。推进：不是所有经验都能升级，必须经过重复出现、验证有效、边界清楚。机制：candidate → verified memory → procedure → skill。落点：skill 是外置大脑把经验转成行动能力的方式。

## 结尾任务

落到：记忆让 AI 少忘，技能让 AI 少问。引出第 12 篇：把经历、触发、验证、生命周期、技能晋升串起来，才是真正的外置大脑。

## 封面提示词

封面大字：AI 不是要记更多 / 是要长技能。封面副标题：经验反复验证后，才该升级成套路。画面：轻漫画、浅底、黑红大字、黄色强调条。AI 从一堆便签中提炼出一本“Skill 手册”；风逆在旁边点头，屏幕上显示“写前卡片 → 初稿 → 评审 → 复审”。黄色便签：“反复有效”“边界清楚”“升级成技能”。中央安全区兼容公众号裁剪。

## 跑偏风险

不要写成插件教程或抽象学习理论。必须从具体失败样本如何变成技能规则进入。

## 评审重点

读者能不能理解“记忆”和“技能”的区别：记忆是知道过去，技能是下一次会更稳定地做对。
